HTTP ticket and HTTP basic use TVHeadend HTTP streaming. HTSP uses TVHeadend's native subscription protocol; when HTSP is selected, TVHeadend HTTP fallback is disabled and playback failures are reported as HTSP errors.
Tvheadend defaults to 500000 bytes. A conservative 2000000-byte queue reduces I/P/B frame drops without the large per-subscription memory cost of Kodi's 10000000-byte setting. Set 0 to use the Tvheadend default.
